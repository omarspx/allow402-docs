## Credit unit

1 credit = 1 micro-USDC = $0.000001 USD.

All amounts in the API and Dashboard are in credits. This keeps numbers readable: 1,000,000 credits = $1.00.

| Credits | USD |
|---------|-----|
| 1 | $0.000001 |
| 1,000 | $0.001 |
| 10,000 | $0.01 |
| 100,000 | $0.10 |
| 1,000,000 | $1.00 |

## Pricing per call

| Scenario | Cost |
|----------|------|
| Target is a free API (no 402) | 1 credit flat |
| Target requires x402 payment | upstream price + 5% commission (min 1 credit) |
| Cache hit (any endpoint) | 1 credit |

The 5% commission is rounded up to the nearest credit, minimum 1.

**Example — x402 API priced at 1,000 credits:**
```
Upstream:     1,000 credits
Commission:      50 credits  (5%)
Total:        1,050 credits  ($0.00000105)
```

## Cache discounts

When you pass `cacheMaxAge > 0`, allow402 applies degressive pricing on cache misses:

| cacheMaxAge | Multiplier | Effect |
|-------------|------------|--------|
| 0 (always fresh) | 1.0× | Full price |
| 1–30 s | 0.5× | 50% off |
| 31–300 s | 0.2× | 80% off |
| 300 s+ | 0.1× | 90% off |

Cache hits always cost 1 credit regardless of the upstream price.

## Top-up methods

### Credit card (Stripe)

Pay in USD via the Dashboard. Credits are available instantly. Minimum $1.

### USDC on Base L2

Send USDC to your allow402 deposit address (shown in Dashboard → Billing). Credits are applied 1:1 on confirmation. Minimum 1 USDC.

## Per-key budgets

Each API key can have two spending controls:

**`allocatedCredits`** — Hard credit cap. The key is locked when `creditsSpent >= allocatedCredits`. Calls return `402 KEY_BUDGET_EXHAUSTED`.

**`maxCostPerCall`** — Per-call cost ceiling. Calls whose upstream x402 price exceeds this value are rejected before contacting the service. Returns `402 CALL_COST_LIMIT_EXCEEDED`.

Both are optional and can be set at key creation time or updated later in the Dashboard.
